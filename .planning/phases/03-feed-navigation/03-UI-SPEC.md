## Typography

Phase 3 is icon/control-dominant. The only new text element is the end-of-feed message. Loading/error state copy inherits Phase 2 patterns.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (`text-base`) | 400 | 1.5 | Feed loading state ("Loading...") |
| Label | 14px (`text-sm`) | 600 | 1.2 | Error state message |
| End-of-feed | 16px (`text-base`) | 400 (`font-normal`) | 1.0 | "You've seen everything" — single centered line |
| — | — | — | — | No heading or display text in feed UI |

**Font stack:** `system-ui, -apple-system, sans-serif` — Phase 2 precedent, unchanged.

**Weights declared:** regular (400) + semibold (600). The end-of-feed copy uses `font-normal` (400) — matches body weight, communicating neutral completion. Visual distinction from body text is provided by `tracking-wide` letter-spacing rather than a third weight. Source: CONTEXT.md D-08; RESEARCH.md EndOfFeedSlide code example.

**Tracking for end-of-feed:** `tracking-wide` — slight letter-spacing on "You've seen everything" for visual breathing room on a clean black screen. Source: RESEARCH.md EndOfFeedSlide code example.

## Interaction Contract

> Phase 3 adds swipe navigation, buffering detection, and end-of-feed lock on top of the Phase 2 player contract. Phase 2 interaction rules (mute button, progress bar, tap-to-pause, pause flash) are unchanged and inherited by reference.

### Swipe Navigation (D-01, D-02, D-03, FEED-01, FEED-02, FEED-03)

- **Direction:** vertical (`direction="vertical"`) — swipe up for next, swipe down for previous
- **Snap behavior:** full snap to next/previous slide; `slidesPerView={1}`, `speed={250}` (200-300ms per D-01)
- **No momentum/rubber-band:** `resistance={false}` — finger release immediately commits (D-01)
- **No overscroll:** edge slides do not bounce; Swiper handles edge boundary natively (D-02)
- **Desktop mousewheel:** `mousewheel={{ forceToAxis: true }}` — one scroll click = one video advance (D-03)
- **Swiper module required:** `Mousewheel` from `swiper/modules`; CSS: `import 'swiper/css'` only (not bundle)
- **Container:** `<div className="fullscreen-container bg-black">` wrapping `<Swiper className="h-full w-full">`
- **Transition visual:** no custom animation; Swiper default snap transition at 250ms

### Preloading (D-04, FEED-04)

- **Active slide video:** `preload="auto"` — browser downloads aggressively for current video
- **Adjacent slides (N+1, N+2, N-1):** `preload="metadata"` — browser fetches first 2-5% for near-instant start
- **Non-adjacent slides:** `preload="none"` — no download until slides approach viewport
- **DOM strategy:** Swiper keeps 3-4 slides in DOM at all times (non-virtual); this is required — do NOT use Swiper Virtual module (destroys `<video>` elements, breaking back-swipe cache)

### Buffering Spinner (D-06, D-07, FEED-05)

- **Trigger:** `waiting` HTML media event fires → show spinner
- **Hide triggers:** `canplay` OR `playing` HTML media event fires → hide spinner (both required per pitfall 6 in RESEARCH.md)
- **Position:** `absolute inset-0 flex items-center justify-center pointer-events-none` — centered over entire slide
- **Spinner element:** 40×40px (`w-10 h-10`) circle, `rounded-full`, `border-4`, `border-white/30`, `border-t-white`, `animate-spin`
- **No text label:** spinner alone — no loading copy (D-06)
- **Same treatment for initial load and mid-playback stall:** single `isBuffering` boolean state handles both (D-07)
- **Pointer events:** `pointer-events-none` on wrapper — spinner never intercepts mute button or tap-to-pause

### Mute State Persistence (D-05)

- **Store:** Zustand `feedStore` holds `isMuted: boolean` — initialized `true` (autoplay muted per PLAY-02)
- **On slide change:** `onSlideChange` syncs new active video's `.muted` property from `useFeedStore.getState().isMuted` (use `getState()` not hook to avoid stale closure)
- **Visual:** same MuteButton from Phase 2 — no change to MuteButton component itself
- **User perception:** unmuting on video 1 → swipe to video 2 → video 2 plays with sound (TikTok behavior)

### End-of-Feed Slide (D-08, D-09, FEED-06)

- **Position in slide list:** last `SwiperSlide` after all video slides — a sentinel slide
- **Layout:** `fullscreen-container bg-black flex items-center justify-center`
- **Text:** "You've seen everything" — centered, single line
- **Text style:** `text-white text-base font-normal tracking-wide`
- **No buttons, no illustrations, no action prompts** (D-08)
- **Forward swipe lock:** `onSlideChange` sets `swiper.allowSlideNext = false` when `activeIndex >= totalVideoCount` (mutate instance, not React prop — see RESEARCH.md Pitfall 1)
- **Back swipe:** allowed — user can swipe back down from end-of-feed slide to previous video (D-09)

### Single Active Video Invariant (MOBL-04)

- **Mechanism:** `onSlideChange` pauses previous video, plays next video
- **Previous video:** `videoRefs.current[prevIndex]?.pause()` then `?.load()` to reset to `currentTime = 0`
- **Next video:** `video.muted = isMuted; video.play().catch(() => {})` — explicit mute sync before play
- **All video elements:** must have `playsInline` attribute (non-negotiable on iOS, MOBL-02)

### Infinite Feed Loading

- **Trigger:** `onSlideChange` when `activeIndex >= totalVideoCount - 3` and `hasNextPage`
- **Action:** `fetchNextPage()` — TanStack Query adds new page; Swiper renders new slides
- **No visual indicator** for background page fetch — new slides appear as user approaches the end
- **Loading state:** standard feed fetch loading shown only on initial app load (inherited from Phase 2 App.tsx loading pattern)

## States Inventory

All states that must have a visual representation in Phase 3:

| State | Visual | Trigger |
|-------|--------|---------|
| Feed loading (initial) | Black screen + "Loading..." centered white text | `useInfiniteQuery` `isPending` |
| Feed error | Centered error text (14px, white, semibold) | `useInfiniteQuery` `isError` |
| Feed empty | "No videos available" + sub-copy | `allVideos.length === 0` |
| Video playing | Video fills screen; mute button; progress bar; no overlay | `playing` event |
| Video buffering | Video (or black) fills screen + centered `animate-spin` white spinner | `waiting` event |
| Video buffering resolved | Spinner disappears; video resumes | `canplay` or `playing` event |
| Swipe in progress | Swiper transition at 250ms — partial overlap of two slides visible | Swiper `onTransitionStart` |
| End-of-feed | Black screen + "You've seen everything" centered white text | `activeIndex === totalVideoCount` |
| Paused (flash) | Pause flash icon (inherited from Phase 2) | tap-to-pause |
| Muted | Muted icon state on MuteButton (inherited) | initial state + toggle |
| Unmuted | Unmuted icon state on MuteButton (inherited) | toggle |

**States inherited from Phase 2 (no change):** playing, paused, paused-flash, paused-clean, muted, unmuted.
