**2. Add auth preHandler hook to `packages/backend/src/routes/feed.ts`:**

Add imports at the top:
```typescript
import { fromNodeHeaders } from 'better-auth/node'
import { auth } from '../auth.js'
```

Add a preHandler hook inside the `feedRoutes` function, BEFORE the existing `fastify.get('/feed', ...)`:
```typescript
fastify.addHook('preHandler', async (request, reply) => {
  const session = await auth.api.getSession({
    headers: fromNodeHeaders(request.headers),
  })
  if (!session) {
    reply.status(401).send({ error: 'Unauthorized' })
  }
})
```

This makes ALL routes registered in the feedRoutes plugin require authentication.

Add `/api` to the proxy configuration:
```typescript
server: {
```

This ensures all `/api/auth/*` requests from the frontend are proxied to the backend during development. Without this, auth calls will fail with CORS errors in development (Research Pitfall 2).

**3. Create `packages/frontend/src/auth/client.ts`:**

```typescript

  baseURL: 'http://localhost:3000',
})
```

The `baseURL` is set explicitly for portability even though the Vite proxy covers `/api` in development. In production, this would point to the actual backend URL.

**4. Update `packages/frontend/src/api/feedApi.ts`:**

Add `credentials: 'include'` to the fetch call:
```typescript
const res = await fetch(`/feed?${params}`, { credentials: 'include' })
```

Without this, the browser will not send the session cookie with feed API requests, and the auth guard will return 401 even for logged-in users (Research Pitfall 7).
