| Token | Value | Usage |

Exceptions:
- **Touch targets:** All interactive controls (mute button) must be at minimum 44×44px per Apple HIG. Achieved via `min-w-[44px] min-h-[44px]` — this is a size constraint, not a spacing token. Source: RESEARCH.md MOBL-03.
- **Progress bar height:** 2px — deliberately sub-scale to stay visually thin. Source: RESEARCH.md Pattern 5 code.
- **Pause flash icon:** 80px (w-20 h-20) circle — sized for center-screen legibility, not spacing grid. Source: RESEARCH.md Pattern 6 code.

## Typography

Phase 2 is an icon-dominant UI. There is no body text or heading text rendered on the video surface. The only typographic element is the pause flash icon (unicode triangle ▶), which is treated as an icon, not text.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Fallback / loading/error states only |
| Label | 14px | 600 | 1.2 | Error message label if shown |
| — | — | — | — | No heading or display text in player UI |

**Font stack:** `system-ui, -apple-system, sans-serif` — no web font loading. Performance and native feel on mobile prioritized.

**Weights declared:** regular (400) + semibold (600). Maximum 2 weights per contract rules.

## Color

The player is a dark, full-screen immersive surface. The video content provides all color — UI controls use white/black at reduced opacity to stay legible over any video content.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#000000` / transparent | Video fills the screen; black is the fallback background before video loads | Default — video-first UI |
| Secondary (30%) | `rgba(0,0,0,0.5)` = `black/50` | Pause flash icon background circle; loading overlay | RESEARCH.md Pattern 6 |
| Accent (10%) | `#FFFFFF` = `white` | Progress bar fill, mute icon, pause triangle icon | Claude's discretion — maximum contrast over video |
| Destructive | not applicable | No destructive actions in Phase 2 | — |

**Accent reserved for:** progress bar fill indicator, mute/unmute button icon, pause flash triangle icon. These are the only 3 elements that use pure white. No other use.

**Progress bar track:** `rgba(255,255,255,0.3)` = `white/30` — subdued track behind the white fill. Source: RESEARCH.md Pattern 5 code (`bg-white/30`).

**Color rationale:** Full-screen video UIs (TikTok, Instagram Reels, YouTube Shorts) universally use white controls at 60-100% opacity over dark video content. Any tinted accent color risks clashing with video frames. White is the correct and standard choice for this UI type.

## Interaction Contract

> This section supplements the color/typography contract with the specific interaction states mandated by CONTEXT.md decisions. Executor MUST implement these exactly.

### Mute Button (D-01, D-02, PLAY-03)

- **Position:** absolute, bottom-right corner of video container
- **Inset:** `bottom-6 right-4` (24px bottom, 16px right) — thumb-reachable zone
- **Always visible:** no auto-hide (D-02). Opacity 1 at all times.
- **Size:** minimum 44×44px touch target (MOBL-03)
- **Tap behavior:** toggles audio only; `e.stopPropagation()` required (D-04)
- **Muted state icon:** speaker with X / slash through it (SVG inline)
- **Unmuted state icon:** speaker with sound waves (SVG inline)
- **Icon color:** white at 100% opacity
- **Icon container:** no background circle; icon alone on video

### Progress Bar (D-03, PLAY-06)

- **Position:** absolute, bottom edge of video container (`bottom-0 left-0 right-0`)
- **Height:** 2px
- **Track color:** `white/30` (`rgba(255,255,255,0.3)`)
- **Fill color:** `white` (100% opacity)
- **Animation:** CSS `transition-[width] duration-100 ease-linear` — smooth 100ms interpolation
- **Always visible:** no auto-hide (D-02). Visible from first frame.
- **Progress source:** `timeupdate` event on `<video>` ref → `currentTime / duration` ratio

### Tap-to-Pause (D-04, PLAY-04)

- **Hit area:** full video container `div` — entire screen except mute button
- **Mute button exclusion:** achieved via `pointer-events-none` on overlay parent + `pointer-events-auto` on mute button + `e.stopPropagation()` on mute tap
- **No visual feedback** on the tap itself — feedback is the pause flash icon only

### Pause Flash Icon (D-05)

- **Trigger:** fires only on transition from playing → paused (not on initial load)
- **Position:** centered absolutely within video container (`absolute inset-0 flex items-center justify-center`)
- **Container:** 80×80px circle, `bg-black/50`, `rounded-full`
- **Icon:** white play triangle ▶, `text-4xl` (36px), `text-white`, `ml-1` (optical centering)
- **Animation timing:**
  - Appears instantly at opacity 1
  - Holds at opacity 1 for 500ms
  - Fades to opacity 0 over 300ms CSS transition
  - Removed from DOM at 800ms (after fade completes)
- **After fade:** video stays paused; screen is clean — no persistent overlay (D-05)
- **Pointer events:** `pointer-events-none` — never intercepts taps

### Video Element Baseline (MOBL-01, MOBL-02, PLAY-01, PLAY-02, PLAY-05)

- **Attributes (mandatory):** `autoPlay muted loop playsInline`
- **Object fit:** `object-cover w-full h-full` — covers full container regardless of aspect ratio (D-06)
- **Container:** `h-dvh w-full` with `-webkit-fill-available` fallback in index.css

## Viewport Contract

| Rule | Implementation | Requirement |
|------|----------------|-------------|
| Full-screen height | `h-dvh` Tailwind utility + `min-height: -webkit-fill-available` CSS fallback | MOBL-01 |
| No scroll | `overflow: hidden` on `html`, `body`, `#root` | Prevents rubber-band scroll revealing white behind video |
| No letterboxing | `object-fit: cover` on `<video>` | PLAY-01, D-06 |
| Address bar avoidance | `100dvh` (dynamic viewport; excludes browser chrome) | MOBL-01 |

CSS in `packages/frontend/src/index.css`:
```css
@import "tailwindcss";

html, body, #root {
  height: 100%;
  margin: 0;
  padding: 0;
  overflow: hidden;
}

.fullscreen-container {
  height: 100dvh;
  min-height: -webkit-fill-available;
  width: 100%;
  overflow: hidden;
}
```

## States Inventory

All states that must have a visual representation, derived from requirements and success criteria:

| State | Visual | Trigger |
|-------|--------|---------|
| Loading (feed fetch) | Black screen — no spinner needed for Phase 2; TanStack Query loading state | `useQuery` `isLoading` |
| Error (feed fetch) | Centered error text (14px, white, semibold) | `useQuery` `isError` |
| Empty feed | Centered "No videos available" text | `data.items.length === 0` |
| Playing | Video visible, no overlay icons, progress bar advancing | `video.play()` succeeds |
| Paused (fresh) | Video visible, no overlay, progress bar static | tap on video body |
| Paused (flash visible) | Video + centered pause flash icon at opacity 1 | 0–500ms after pause |
| Paused (flash fading) | Video + centered pause flash icon at opacity 0 fading | 500–800ms after pause |
| Paused (clean) | Video only, no icons visible | > 800ms after pause |
| Muted | Mute icon shows "muted" state | initial state (PLAY-02) |
| Unmuted | Mute icon shows "sound on" state | after first unmute tap |
